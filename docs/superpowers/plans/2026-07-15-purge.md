## Global Constraints

- API change is ADDITIVE only; API.md gets the endpoint doc + freeze-header version note (exact text in spec). No MCP purge tool. No change to existing routes/shapes.
- `through_seq` is INCLUSIVE (`seq <= N`). All-purged → session row deleted, `session: null` returned.
- One transaction: delete + recompute (`event_count/error_count/warn_count/first_ts/last_ts/sources` from survivors; label kept).
- UI purge only via chip + `window.confirm`. On success: drop `after` from URL, refetch; `session: null` → navigate `/`.
- TDD; no trailers; commit per task.

### Task 1: Server endpoint (TDD)

**Files:** Modify `packages/server/src/queries.ts` (+`purgeEventsThrough`), `packages/server/src/app.ts` (+route), `API.md`. Test: `packages/server/test/queries.test.ts` (+purge unit tests), `packages/server/test/api.test.ts` (+route tests).

**Interfaces:** `purgeEventsThrough(db: Db, sessionId: string, throughSeq: number): { deleted: number; sessionDeleted: boolean }` — returns deleted row count and whether the session row was removed (no survivors). Route: parse `through_seq` via the existing `num()` helper (`400` JSON if undefined/non-finite); unknown session → `404` JSON (check BEFORE purging); else `200 { deleted, session }` where `session` = `getSession(db, id, now())` post-purge or `null` when `sessionDeleted`.

Unit tests (fixture: s1 with mixed levels/sources across seqs, plus a second session s2): boundary — `through_seq` = mid seq deletes `<=` only; counters/first_ts/last_ts/sources recomputed to survivors exactly; all-purged → `sessionDeleted: true`, session row gone, events gone; s2 completely untouched (events + counters); `through_seq` below min seq → deleted 0, session unchanged. Route tests: 200 shape (deleted + updated session), `session: null` case, 404 unknown, 400 missing/garbage `through_seq`, and existing `DELETE /api/sessions/:id` unaffected.

API.md: endpoint section mirroring the spec (params, semantics incl. inclusive naming rationale, response table, all-purged behavior) + version note. Commit `feat(server): purge — DELETE events through a seq with counter recompute`.
